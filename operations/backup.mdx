TDengine provides two backup approaches: **taosdump** for logical backup and restore (all editions), and **incremental backup via data subscription** (Enterprise only). Choose based on your recovery point objectives and whether you need point-in-time restore.

## taosdump

taosdump is an open-source tool bundled with TDengine. It exports data in [Apache Avro](https://avro.apache.org/) format, which is portable and can be used to migrate data between clusters or to external ecosystems.

### Full backup

Back up a single database to a directory:

```bash
taosdump -h localhost -P 6030 -D dbname -o /backup/path
```

Back up multiple databases:

```bash
taosdump -h localhost -P 6030 -D db1,db2 -o /backup/path
```

Back up all databases:

```bash
taosdump -h localhost -P 6030 -A -o /backup/path
```

Back up specific supertables or tables within a database:

```bash
taosdump -h localhost -P 6030 dbname stb1 stb2 tbname1 -o /backup/path
```

<Warning>
If the output directory already contains backup files, taosdump exits with an error to prevent overwriting. Use a unique directory for each backup run.
</Warning>

### Incremental / time-range backup

taosdump does not support checkpointing. If a backup is interrupted, delete all partial output and restart. For large databases, split the backup by time range using `-S` and `-E`:

```bash
taosdump -h localhost -P 6030 -D dbname \
  -S 2024-01-01T00:00:00+00:00 \
  -E 2024-06-30T23:59:59+00:00 \
  -o /backup/2024-h1

taosdump -h localhost -P 6030 -D dbname \
  -S 2024-07-01T00:00:00+00:00 \
  -E 2024-12-31T23:59:59+00:00 \
  -o /backup/2024-h2
```

### Schema-only backup

Export only the table and supertable schemas without data:

```bash
taosdump -h localhost -P 6030 -D dbname -s -o /backup/schema-only
```

### Key options

| Option | Description |
|--------|-------------|
| `-h HOST` | Server hostname (default: `localhost`) |
| `-P PORT` | Server port (default: `6030`) |
| `-u USER` | Username (default: `root`) |
| `-D db1,db2` | Comma-separated list of databases to back up |
| `-A` | Back up all databases |
| `-o PATH` | Output directory |
| `-i PATH` | Input directory (for restore) |
| `-S TIME` | Start time (ISO8601 or epoch) |
| `-E TIME` | End time (ISO8601 or epoch) |
| `-s` | Schema only, no data |
| `-B NUM` | Rows per query/insert batch (default: 16384) |
| `-T NUM` | Number of threads (default: 8) |
| `-d CODEC` | Avro compression: `null`, `deflate`, `snappy`, `lzma` |
| `-Z DRIVER` | Connection driver: `Native` (default) or `WebSocket` |

### Restore

Restore from a backup directory to a running cluster:

```bash
taosdump -h localhost -P 6030 -i /backup/path
```

Restore to a different database name:

```bash
taosdump -h localhost -P 6030 -i /backup/path -W "olddb=newdb"
```

<Note>
Restore supports interruption recovery — if the process is killed, restart it in the same output directory and taosdump skips tables that already exist. You may see "table already exists" messages, which you can safely ignore.
</Note>

If you encounter a `WAL size exceeds limit` error during restore, reduce the batch size:

```bash
taosdump -h localhost -P 6030 -i /backup/path -B 1000
```

## Enterprise: incremental backup

TDengine Enterprise implements incremental backup using data subscription. Each backup cycle generates a **backup point** that captures all data changes (inserts, modifications, deletes, metadata changes) since the previous point.

### Concepts

| Term | Meaning |
|------|---------|
| Backup object | A database or supertable |
| Backup plan | A periodic task with a start time and cycle interval |
| Backup point | A set of backup files generated by one backup task run |
| Initial backup point | The first backup point; restore always starts here |
| Restore task | Replays backup points from the initial point to a target point |

### Creating a backup plan

<Steps>
  <Step title="Open taosExplorer">
    Navigate to `http://your-host:6060` in a browser.
  </Step>
  <Step title="Go to backup settings">
    In the left menu, click **Management → Backup**.
  </Step>
  <Step title="Create a new backup plan">
    Click **Create New Backup** and fill in:

    - **Database**: The database to back up.
    - **Super Table**: (Optional) Restrict backup to a specific supertable.
    - **Next execution time**: When to run the first backup. A past date triggers an immediate backup.
    - **Backup Cycle**: How often to run incremental backups. Must be less than `WAL_RETENTION_PERIOD` for the database.
    - **Retry times**: Number of retries on failure.
    - **Retry interval**: Seconds between retries.
    - **Directory**: Full path to store backup files.
    - **Backup file max size**: Maximum size of a single backup file. Larger backups are split automatically.
    - **Compression level**: `fastest`, `best`, or `balanced`.
  </Step>
</Steps>

<Warning>
The backup cycle must be shorter than `WAL_RETENTION_PERIOD` for the backed-up database. If you see `[0x038C] WAL retention period is zero`, set it:

```sql
ALTER DATABASE mydb WAL_RETENTION_PERIOD 3600;
```
</Warning>

### S3 object storage (Enterprise)

Enable S3 dumping to upload backup files to object storage automatically. Provide:

- **Endpoint**: S3-compatible endpoint URL
- **Access Key ID** and **Secret Access Key**: Authentication credentials
- **Bucket**: Target bucket name
- **Region**: Bucket region
- **Object Prefix**: A prefix (directory-like path) for backup objects in the bucket
- **Backup Retention Period**: Local retention window; files older than `now - retention_period` must be uploaded to S3 before they are eligible for local deletion
- **Backup Retention Count**: Number of most-recent backup files to keep locally

### Restoring from a backup point

<Steps>
  <Step title="Locate the backup plan">
    In **Management → Backup**, find the plan and click **View** in the **Backup File** column.
  </Step>
  <Step title="Choose a backup point">
    Select the backup point (timestamp) to restore to, then click the **Restore** icon.
  </Step>
  <Step title="Confirm restore">
    Select the target database and click **Confirm**. The restore task replays all backup points from the initial point to the selected timestamp.
  </Step>
</Steps>

## Backup best practices

<CardGroup cols={2}>
  <Card title="Test restores regularly">
    Run a restore to a non-production environment periodically to verify backup integrity. A backup you have never tested is not a backup.
  </Card>
  <Card title="Use time-range segmentation">
    For large databases, use `-S` and `-E` to split taosdump backups into manageable chunks. This also makes restoring a specific time window easier.
  </Card>
  <Card title="Store backups off-host">
    Copy taosdump output to a separate server or object storage. On-host backups do not protect against disk failure.
  </Card>
  <Card title="Set WAL_RETENTION_PERIOD">
    For Enterprise incremental backup, ensure `WAL_RETENTION_PERIOD` is set long enough to cover at least one backup cycle plus a safety margin.
  </Card>
</CardGroup>

## Troubleshooting

### Port connection error

```text
Error: tmq to td task exec error
Caused by: [0x000B] Unable to establish connection
```

Check that the source FQDN is resolvable and port 6030 is open and listening.

### WebSocket name resolution error

```text
0: WebSocket internal error: IO error: failed to lookup address information: Temporary failure in name resolution
```

- `Temporary failure in name resolution`: The FQDN cannot be resolved. Check DNS or `/etc/hosts`.
- `IO error: Connection refused (os error 111)`: The port is closed. Verify taosAdapter is running on the configured port.
- `received corrupt message`: The server may not support TLS on the configured port.
