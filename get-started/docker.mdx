---
title: "Deploy with Docker"
description: "Run TDengine in a Docker container, connect with the taos CLI, and run your first queries."
---

You can install TDengine TSDB in a Docker container and perform basic tests to verify its performance. This is the fastest way to get TDengine running locally.

<Note>
  To install TDengine directly on your machine instead, see [Deploy from Package](/get-started/package).
</Note>

## Prerequisites

- [Docker](https://www.docker.com/) installed and running on your machine.
- Ensure the ports required by TDengine are not already in use: `6030`, `6041`, and `6043`–`6060` (TCP and UDP).

## Set up the container

<Steps>
  <Step title="Pull the TDengine image">
    Pull the latest TDengine Enterprise image from Docker Hub:

    ```bash
    docker pull tdengine/tsdb-ee:latest
    ```

    <Tip>
      To pin a specific version, replace `latest` with the version tag, for example `tdengine/tsdb-ee:3.3.6.0`.
    </Tip>
  </Step>

  <Step title="Start the container">
    Run TDengine with all required ports exposed:

    ```bash
    docker run -d \
      -p 6030:6030 \
      -p 6041:6041 \
      -p 6043-6060:6043-6060 \
      -p 6043-6060:6043-6060/udp \
      tdengine/tsdb-ee
    ```

    To persist data and logs to your local machine, mount host directories into the container:

    ```bash
    docker run -d \
      -v <local-data-directory>:/var/lib/taos \
      -v <local-log-directory>:/var/log/taos \
      -p 6030:6030 \
      -p 6041:6041 \
      -p 6043-6060:6043-6060 \
      -p 6043-6060:6043-6060/udp \
      tdengine/tsdb-ee
    ```
  </Step>

  <Step title="Verify the container is running">
    Confirm the container started successfully:

    ```bash
    docker ps
    ```

    You should see `tdengine/tsdb-ee` listed with a status of `Up`.
  </Step>

  <Step title="Open a shell inside the container">
    Replace `<container-name>` with the name shown in `docker ps`:

    ```bash
    docker exec -it <container-name> bash
    ```
  </Step>

  <Step title="Launch the taos CLI">
    Inside the container, start the TDengine command-line interface:

    ```bash
    taos
    ```

    You will see the following prompt when the CLI is ready:

    ```sql
    taos>
    ```
  </Step>
</Steps>

## Run your first queries

Each SQL statement in the taos CLI must end with a semicolon (`;`).

### Create a database and table

```sql
CREATE DATABASE demo;
USE demo;
CREATE TABLE t (ts TIMESTAMP, speed INT);
```

### Insert data

```sql
INSERT INTO t VALUES ('2019-07-15 00:00:00', 10);
INSERT INTO t VALUES ('2019-07-15 01:00:00', 20);
```

### Query the data

```sql
SELECT * FROM t;
```

Expected output:

```sql
           ts            |    speed    |
========================================
 2019-07-15 00:00:00.000 |          10 |
 2019-07-15 01:00:00.000 |          20 |

Query OK, 2 row(s) in set (0.003128s)
```

## Benchmark with taosBenchmark

`taosBenchmark` is a tool for evaluating TDengine's ingestion, query, and subscription performance. Run it with a single command to generate 100 million records:

```shell
taosBenchmark -y
```

This creates a database named `test` containing a supertable `meters` with 10,000 subtables (`d0`–`d9999`), each with 10,000 records and fields for `ts`, `current`, `voltage`, and `phase`.

After the data is loaded, run these queries in the taos CLI to explore the dataset:

```sql
-- Count all records
SELECT COUNT(*) FROM test.meters;

-- Average, max, and min across all records
SELECT AVG(current), MAX(voltage), MIN(phase) FROM test.meters;

-- Filter by tag value
SELECT COUNT(*) FROM test.meters WHERE location = "California.SanFrancisco";

-- Aggregate in 10-second windows for a single device
SELECT _wstart, AVG(current), MAX(voltage), MIN(phase)
FROM test.d1001
INTERVAL(10s);
```

<Note>
  The pseudocolumn `_wstart` represents the start time of each aggregation window in interval queries.
</Note>

## Access TDengine Explorer

TDengine TSDB Explorer is a web-based interface for managing databases, running SQL, and exploring data. With TDengine running, open your browser and navigate to:

```
http://localhost:6060
```

Log in with the default credentials: username `root`, password `taosdata`.
